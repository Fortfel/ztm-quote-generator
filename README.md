Project is built using TypeScript with ESLint for code linting
and TypeDoc for generating documentation.

## Getting Started

### Prerequisites

Make sure you have the following installed:

- Node.js (version 14.x or higher)
- npm (comes with Node.js)

### Installation

1. Clone the repository:

   ```
   git clone https://github.com/your-username/your-project.git
   cd your-project
   ```

1. Install the project dependencies:

   ```
   npm install
   ```

1. Setup in your IDE Eslint and Prettier

### Project Structure

```
├── src                 # Source code for the project
├── dist                # Compiled JavaScript files (generated)
├── docs                # Documentation (generated by TypeDoc)
├── tsconfig.json       # TypeScript configuration
├── .eslintrc.json      # ESLint configuration
└── package.json        # Project metadata and dependencies (include Typedoc configuration)
```

## Development

### Running the Project

To compile the TypeScript files to JavaScript:

```
npm run build:ts
```

To automatically recompile the TypeScript files when changes are made:

```
npm run watch:ts
```

### Linting the Code

ESLint is used to ensure code quality and enforce style guidelines.
Run the following command to lint your TypeScript files:

```
npm run lint
npm run lint:fix
```

### Generating Documentation

This project uses **TypeDoc** to generate API documentation from the TypeScript code.
The documentation will be generated in the docs folder:

```
npm run docs
```

To continuously generate documentation as you code, use the watch mode:

```
npm run watch:docs
```

## Scripts

The project includes several npm scripts to automate tasks:

- `npm run build:ts`: Compiles the TypeScript files.
- `npm run watch:ts`: Compiles the TypeScript files in watch mode.
- `npm run lint`: Lints the TypeScript code using ESLint.
- `npm run lint:fix`: Automatically fixes any fixable linting errors.
- `npm run docs`: Generates documentation using TypeDoc.
- `npm run watch:docs`: Watches for changes and regenerates TypeDoc documentation.

## ESLint Configuration

ESLint is set up to lint TypeScript code. You can find the configuration
in the `.eslintrc.js` file. Some useful commands:

- `npm run lint`: Lint all TypeScript files.
- `npm run lint:fix`: Automatically fix any fixable linting errors.

## TypeDoc Configuration

TypeDoc is configured in the `package.json` file.
It generates API documentation based on your TSDoc comments in the source code.

- Documentation will be output to the `docs` folder.
- To generate the docs, run:

```
npm run docs
```
